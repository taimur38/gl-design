04 — Spacing
Tight coupling, generous separation
One principle: elements that belong together are close, elements that don't are far apart. All spacing values derive from the baseline.

The logic

Headings — Large space above (breathing room between sections), small space below (coupling to content they introduce).

Paragraphs — 6pt space after each. No first-line indent. Clean block separation.

Figures — 6pt after the last paragraph, then 12pt above the figure title. Title and image are flush (0pt gap). 6pt below caption. The figure group reads as one unit.

Line height

1.36
11pt text on 15pt leading
= 0.208" per baseline unit

More generous than 1.15 (cramped for long-form), tighter than 1.5 (wastes space). This single value cascades into the entire vertical system: baseline → vertical module → figure heights → spacing values.

06 — Principles
Why this is a system, not a style guide
A style guide lists rules. A system has them derive from each other, so changing one root value keeps everything coherent.

Everything derives from something

Type scale from 1.25 ratio. Vertical modules from line height. Figure heights from modules. Spacing from baselines. Column widths from live area. Change one root and the system adapts.

Researchers skip the design decisions

Pick a figure name, not a pixel size. Set the theme, not the fonts. Choose a palette by sector. Markdown in, styled PDF out. The framework absorbs complexity.

Charts are native to the document

The ggplot theme uses the same two fonts, the same color tokens, the same weight conventions. Charts don't look pasted on — they look like they grew there.

Same code, two contexts

theme_gl(mode="slide") renders titles inside charts. mode="report" suppresses them for the document to handle. One codebase serves presentations and publications.
