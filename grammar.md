# Growth Lab Visual Grammar

The grammar is the medium-agnostic foundation of the Growth Lab design system:
the color, type, and chart conventions that hold true whether an output is a
50-page report, a presentation slide, a one-page brief, a standalone chart, or
a web page. Specific outputs ("recipes") apply the grammar to a medium —
picking page geometry, anchor sizes, and density to suit the artifact. See
[`recipes/`](recipes/) for those applications.

## 2. Typography

### Type stack

Two families do the entire job:

- **Source Sans 3** — body text, headings, anything meant to be read. Open,
  legible, neutral. The default voice of the lab.
- **JetBrains Mono** — captions, axis labels, table notes, technical
  annotations, page running heads. Monospace signals "data" and gives a quiet
  visual separation between prose and machine output.

Both ship locally in [`assets/fonts/`](assets/fonts/) (no system install
needed); the in-chart theme also pulls them from Google Fonts at runtime.

### Type scale (the system)

A **type scale** is a small set of sizes generated by repeatedly multiplying
a base size by a fixed ratio. Committing to a ratio — rather than picking each
size by hand — produces consistent proportional jumps that the eye reads as
visual rhythm.

Growth Lab uses a **×1.25 ratio** ("major third" in the musical naming
convention designers borrow). Calm but clearly stepped; suits research
publication better than the more dramatic 1.333 or 1.5 ratios used in
editorial design.

```
×1.25 → each step is 25% larger than the one below it

step  ratio          example at 11pt anchor
─────────────────────────────────────────────
+3    1.25³ = 1.953  21.5pt   (H2 / section title)
+2    1.25² = 1.563  17pt     (H3 / subsection)
+1    1.25¹ = 1.250  14pt     (H4 / sub-sub)
 0    1.0            11pt     (body)
−1    1.25⁻¹= 0.800  9pt      (footnote / table small)
−2    1.25⁻² = 0.640 8pt      (page header / footer)
```

The grammar commits to **which ratio**; each recipe picks the **anchor** (its
body size) and walks the scale from there. A report anchors at 11pt → 11/14/17/21.5.
A slide deck would anchor higher (~18pt) → 18/22.5/28/35. Same harmonic system,
different absolute values — and the visual identity stays coherent across media
because the *ratio* is what the eye perceives.

### Role hierarchy

Every recipe assigns sizes to a common set of text roles. The roles themselves
(not their sizes) are part of the grammar:

| Role            | Family          | Weight        | Color        |
|-----------------|-----------------|---------------|--------------|
| H1 (title)      | Source Sans 3   | SemiBold      | text_dark    |
| H2 (section)    | Source Sans 3   | SemiBold      | text_dark    |
| H3 (subsection) | Source Sans 3   | SemiBold      | text_dark    |
| H4 (sub-sub)    | Source Sans 3   | SemiBold      | text_dark    |
| Body            | Source Sans 3   | Regular       | text_dark    |
| Body emphasis   | Source Sans 3   | SemiBold      | text_dark    |
| Block quote     | Source Sans 3   | Regular Italic| text_muted   |
| Figure title    | Source Sans 3   | SemiBold      | text_dark    |
| Figure caption  | JetBrains Mono  | Regular       | text_muted   |
| Table header    | Source Sans 3   | SemiBold      | text_dark    |
| Table body      | Source Sans 3   | Regular       | text_dark    |
| Table note      | JetBrains Mono  | Regular       | text_muted   |
| Footnote        | Source Sans 3   | Regular       | text_muted   |
| Page header     | JetBrains Mono  | Regular       | text_muted   |
| Page footer     | JetBrains Mono  | Regular       | text_muted   |

Recipes pin the absolute size of each role at their chosen anchor.

## 3. In-chart typography

Inside a chart, sizes are relative to the figure's own coordinate space (set
via `base_size` in `theme_gl()`), not the page. These ratios are part of the
grammar and should hold across every recipe — a chart that lands in a report
or on a slide reads the same way internally.

| Element       | Family         | Relative size | At base 12pt |
|---------------|----------------|---------------|---------------|
| Plot title    | Source Sans 3  | 1.35×         | 16.2pt        |
| Plot subtitle | Source Sans 3  | 1.0×          | 12pt          |
| Axis title    | JetBrains Mono | 0.85×         | 10.2pt        |
| Axis text     | Source Sans 3  | 0.85×         | 10.2pt        |
| Legend title  | JetBrains Mono | 0.8×          | 9.6pt         |
| Legend text   | Source Sans 3  | 0.85×         | 10.2pt        |
| Strip text    | JetBrains Mono | 0.85×         | 10.2pt        |
| Caption       | JetBrains Mono | 0.75×         | 9pt           |

These are already encoded in `theme_gl()` and should not be overridden
per-chart. See [`skills/gl-ggplot/SKILL.md`](skills/gl-ggplot/SKILL.md).

### Slide vs. report mode

`theme_gl()` accepts a `mode` parameter:

- **`mode = "slide"`** (default) — plot title, subtitle, and caption render
  inside the chart. Use for standalone slides and presentations.
- **`mode = "report"`** — title, subtitle, and caption are suppressed (set to
  `element_blank()`). The surrounding document handles those via Figure Title
  and Figure Source styles instead. Axis labels are kept. Legend goes
  bottom-left by default to preserve plot width.

The same `labs(title, subtitle, caption)` calls work in both contexts — the
theme controls whether they render.
