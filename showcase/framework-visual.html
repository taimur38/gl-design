From baseline to vertical module

Vertical rhythm starts with the body text. Leading becomes a baseline, six baselines become a module, and that module governs every vertical measurement that follows.

11pt body × 1.36 leading = 15pt line-height
15pt ÷ 72 pt/in = 0.208" per baseline
0.208" × 6 baselines = 1.25" vertical module
Six is the smallest count of baselines that reads as a coherent block of text — large enough to feel like a unit, small enough that figures can snap to meaningful multiples of it. From this one decision, every vertical measurement in the framework descends.

The page, in modules

Just as the horizontal grid divides the 6.5" live width into 6 columns of 0.944", the vertical system divides the 8.75" live height into exactly 7 modules of 1.25". The slightly larger 1.25" bottom margin — a convention borrowed from classical editorial design — optically centers the text block and leaves room for page numbers and footnotes.

Grid and modules, fused

Every element on a page occupies a 2D cell: a column span by a module span. Headings, body paragraphs, figure titles, figures, and captions all obey the same grammar — pick a cell, place the element, and the page composes itself.

Layouts built on the same grid

The grammar stays constant — heading sits in its own module, figures couple with their titles, the caption takes a full-width row below the figure, and body text only resumes on a new row after the caption. Content changes; the rhythm doesn't.

A · full figure under H1
H1 opens the section. A full-width figure (rows 3–5) with caption on row 6 below. Body resumes on row 7 — never beside the caption.
B · paired halves under H2
H2 subhead. Two half-width figures share rows 3–5, each with its own title above. Captions line up in row 6, body returns below.
C · body-first, figure below
H2 opens with a longer body span (rows 2–3). The figure takes rows 4–6 and the caption closes the page on row 7 — no body runs beside it.

04 — Spacing
Tight coupling, generous separation
One principle: elements that belong together are close, elements that don't are far apart. All spacing values derive from the baseline.
24pt

6pt

Source: World Bank WDI.

The logic

Headings — Large space above (breathing room between sections), small space below (coupling to content they introduce).

Paragraphs — 6pt space after each. No first-line indent. Clean block separation.

Figures — 6pt after the last paragraph, then 12pt above the figure title. Title and image are flush (0pt gap). 6pt below caption. The figure group reads as one unit.

Line height

1.36
11pt text on 15pt leading
= 0.208" per baseline unit

More generous than 1.15 (cramped for long-form), tighter than 1.5 (wastes space). This single value cascades into the entire vertical system: baseline → vertical module → figure heights → spacing values.

06 — Principles
Why this is a system, not a style guide
A style guide lists rules. A system has them derive from each other, so changing one root value keeps everything coherent.

Everything derives from something

Type scale from 1.25 ratio. Vertical modules from line height. Figure heights from modules. Spacing from baselines. Column widths from live area. Change one root and the system adapts.

Researchers skip the design decisions

Pick a figure name, not a pixel size. Set the theme, not the fonts. Choose a palette by sector. Markdown in, styled PDF out. The framework absorbs complexity.

Charts are native to the document

The ggplot theme uses the same two fonts, the same color tokens, the same weight conventions. Charts don't look pasted on — they look like they grew there.

Same code, two contexts

theme_gl(mode="slide") renders titles inside charts. mode="report" suppresses them for the document to handle. One codebase serves presentations and publications.
